1. INTRODUCTION

Project Title

MB-MOTORS: Premium Vehicle Rental Platform with KYC Verification Gate

Project Overview

MB-MOTORS is a modern, user-friendly vehicle rental platform that integrates identity verification (KYC - Know Your Customer) as a mandatory prerequisite before any booking. This ensures enhanced security, reduced fraud, and compliance with regulatory requirements.

Key Feature: Every user must complete a comprehensive 5-step identity verification process before accessing the rental checkout system. This creates a trustworthy ecosystem for both vehicle owners and renters.

Why KYC?

  • Prevents fraudulent bookings and vehicle misuse
  • Ensures legal compliance and regulatory adherence
  • Builds trust in the platform for both users and vehicle owners
  • Enables better risk assessment and claim resolution

2. OBJECTIVES

Primary Objectives

  1. Secure Rental Platform: Provide a secure vehicle rental marketplace with integrated identity verification to minimize fraud and unauthorized vehicle usage.
  2. Comprehensive KYC Process: Implement a robust 5-step identity verification system accepting government-issued documents (Aadhaar, PAN, Driving License, Passport, Voter ID) and liveness verification.
  3. User Authentication: Develop secure authentication mechanisms using JWT tokens and bcrypt password hashing for user account security.
  4. Seamless Booking Experience: Create an intuitive checkout flow with optional add-ons and multiple payment methods (Card & UPI).
  5. Owner Management: Enable vehicle owners to list their cars with automated earnings calculator.
  6. GPS Tracking: Provide real-time fleet tracking with vehicle status monitoring and trip logs.
  7. Data Security & Privacy: Ensure all uploaded identity documents are encrypted at rest and served privately with proper access controls.

3. EXISTING SYSTEM & LIMITATIONS

Current Vehicle Rental Systems

Most existing vehicle rental platforms operate with minimal verification processes, relying on basic email/phone authentication and simple identity checks.

Key Limitations

  • High Fraud Risk: Limited identity verification leads to fraudulent bookings, vehicle theft, and damage disputes.
  • Lack of Trust: Vehicle owners hesitant to list premium vehicles due to security concerns.
  • Manual Verification: Most require manual document submission and verification, causing delays and bottlenecks.
  • No Real-time Tracking: Inability to monitor vehicles during rental period increases liability.
  • Payment Processing Issues: Limited payment options and no built-in security for transactions.
  • Compliance Gaps: Non-compliance with KYC/AML regulatory requirements in many jurisdictions.
  • Poor User Data Security: Inadequate encryption and security protocols for sensitive documents.
Problem Statement: There is a need for a comprehensive, secure, and user-friendly vehicle rental platform with mandatory identity verification to ensure safe transactions, regulatory compliance, and trust between all stakeholders.

4. PROPOSED SYSTEM

Solution Overview

MB-MOTORS introduces an innovative, automated KYC-gated vehicle rental platform that mandates comprehensive identity verification before any booking, ensuring maximum security and regulatory compliance.

Key Improvements

  • Automated 5-Step KYC Process: Personal info, Aadhaar/PAN verification, Driving License, selfie liveness check, and consent review.
  • Real-time Verification: Integration with document verification APIs for instant validation.
  • Secure Document Storage: Encrypted storage of sensitive documents with private access controls.
  • GPS Tracking: Real-time vehicle location tracking with live fleet map.
  • Multiple Payment Options: Card and UPI integration for flexible checkout.
  • Flexible Add-ons: Insurance, GPS tracker, additional driver, roadside assistance options.
  • Owner Dashboard: Simple listing form with automated earnings calculator.
  • JWT-based Security: Stateless authentication with token-based access control.
Competitive Edge: The mandatory KYC gate, combined with GPS tracking and secure payment processing, positions MB-MOTORS as the most trustworthy and secure rental platform.

6. SYSTEM MODULES & ARCHITECTURE

Module Breakdown

1. User Module

Role: Handles user registration, login, profile management, and user-specific data persistence. Features email/password and OAuth (Google) authentication.

2. Authentication Module

Role: JWT token generation, validation, password hashing with bcryptjs, session management, and protected route middleware.

3. KYC Module (Core)

Role: 5-step identity verification process including document upload, Aadhaar/PAN verification, driving license validation, liveness check (selfie), and final approval workflow.

4. Booking Module

Role: Handles rental booking creation, modification, cancellation, and status tracking. Enforces KYC requirement before booking.

5. Payment Module

Role: Integrates payment gateways (Card & UPI), processes transactions, handles refunds, and maintains payment records.

6. Fleet & Listing Module

Role: Vehicle owners can list vehicles, manage availability, calculate earnings, track bookings, and update vehicle details.

7. GPS Tracking Module

Role: Real-time vehicle location tracking, route history, status updates, and live fleet map visualization.

8. File Upload & Storage Module

Role: Secure document upload using Multer, encryption at rest, private file serving, and access control management.

7.A SYSTEM DIAGRAMS - ER DIAGRAM

Entity Relationship Diagram (ER)

Significance: The ER diagram illustrates the database structure, showing relationships between User, Booking, Vehicle, KYC, and Payment entities. It helps understand data flow and dependencies.

📊 ER Diagram Entities:

  • User: Stores user profile, authentication info, and embedded KYC subdocument
  • KYC (Embedded in User): Personal info, document references, verification status, timestamps
  • Booking: Links User ↔ Vehicle, rental dates, payment reference, add-ons, status
  • Vehicle: Car details, owner info, availability, pricing, location
  • Payment: Transaction records, method, amount, status, reference
Key Relationship: User → KYC (one-to-one embedded) | User → Booking (one-to-many) | Vehicle → Booking (one-to-many) | Booking → Payment (one-to-one)

7.E BOOKING FLOW & API ENDPOINTS

User Journey - Booking Flow

Browse Fleet ↓ [Login Gate] → not logged in → Login/Register ↓ [KYC Gate] → not verified → Complete 5-Step KYC ↓ Checkout Page (Driver details, Add-ons, Payment) ↓ Order Confirmation & Receipt ↓ GPS Tracking & Trip Management

Key API Endpoints

Method Endpoint Purpose
POST /api/auth/register Create new user account
POST /api/auth/login Authenticate user with JWT
POST /api/kyc/submit Submit KYC documents
GET /api/kyc/status Check KYC verification status
POST /api/bookings Create booking (requires verified KYC)
GET /api/bookings List user bookings
PUT /api/bookings/:id/cancel Cancel booking and request refund
